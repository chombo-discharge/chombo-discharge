1 Using VisIt on clusters {#visualization-cluster}
2 =======================
3 
4 To use VisIt's parallel computing capabilities to its fullest, its best to run it in server mode. To do this, _you should have matching versions of VisIt both locally and remotely_.
5 
6 To set up remote visualization, you will need
7 
8 1. The host address.
9 2. The path to VisIt on the server
10 3. Optionally have to modify your .bashrc (or equivalent) so that VisIt is loaded on ssh login.
11 
12 Below, we show how to set up a host profile for fram.sigma2.no
13 
14 Remote visualization on fram.sigma2.no
15 --------------------------------------
16 
17 We will use the remote 2.13.0 remote installation of VisIt on fram. Firstly, modify your .bashrc file so that your login automatically loads the necessary modules. On fram, this is done by appending the following line to your .bashrc file
18 
19  module load VisIt/2.13.0-intel-2017a
20 
21 Next, we will set up the host profile on your local installation. Run VisIt2.13.0 locally
22 
23 1. Go to Options->Host profiles
24 2. Create a new host profile
25 3. In the field 'Host name', use any host name.
26 4. In the field 'Remote host name', type 'fram.sigma2.no'
27 5. In the field 'Path to VisIt installation', type '/cluster/software/VisIt/2.13.0-intel-2017a'
28 6. Check the 'Tunnel data connections through SSH' flag.
29 7. Navigate to the 'Launch profiles' tab.
30 8. Create a new profile and navigate to the 'Parallel' tab.
31 9. In the 'Parallel' tab, check the 'Launch parallel engine' box.
32 
33 To open a file remotely. Select 'Open' and choose your new host under the 'Host' dropdown menu. You will be prompted for a password, after which you can navigate to your remote file.
