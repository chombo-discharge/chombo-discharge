Write an SoA particle container to an H5Part file (quick visualization).

Position, id, and weight are taken from the container-owned columns; the caller supplies the payload datasets as (name, accessor) pairs so the scalar-vs-vector distinction and naming are explicit (the SoA columns tuple interleaves scalar and vector-component columns, so there is no implicit reals/vects partition to rely on). Each vector dataset is written as name-x/name-y/name-z.

Parameters
[in]a_filenameFile name.
[in]a_particlesSoA particle container.
[in]a_scalarVarsScalar datasets: (name, accessor) pairs.
[in]a_vectorVarsVector datasets: (name, accessor) pairs.
[in]a_shiftParticle position shift.
